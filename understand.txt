uvicorn[standard] is the library to actually run fastapi apps 
With Uvicorn, it listens on a network port (e.g. localhost:8000) so that browsers or apps can connect to it.

    exmaple

            from fastapi import FastAPI

            app = FastAPI()

            @app.get("/")
            def home():
                return {"message": "Hello, World!"}

        In terminal run: uvicorn main:app --reload

    In browser: http://127.0.0.1:8000/ ‚Üí you‚Äôll see {"message": "Hello, World!"}.

    http://127.0.0.1:8000/docs ‚Üí you‚Äôll see interactive API docs auto-generated by FastAPI.

    If you want a different port or make it visible to other devices on your network:
        uvicorn main:app --host 0.0.0.0 --port 9000 --reload
    --host 0.0.0.0 = listen on all network interfaces (so other devices on WiFi can reach it).
    --port 9000 = run on port 9000 instead of 8000.

Pydantic: 
    Validating incoming JSON data ‚Üí check that the request has the right fields and types.
    Parsing (converting) data ‚Üí turn input values into the right Python types.
    Error reporting ‚Üí if the data is wrong, return a clear error automatically.

    example:
        from fastapi import FastAPI
        from pydantic import BaseModel

        app = FastAPI()

        # ‚úÖ Define the data model with Pydantic
        class User(BaseModel):
            name: str
            age: int

        # ‚úÖ FastAPI uses Pydantic to validate automatically
        @app.post("/hello")
        def say_hello(user: User):
            return {"message": f"Hello {user.name}, you are {user.age} years old"}

üîπ So what part is exactly Pydantic‚Äôs job?
    class User(BaseModel): ... ‚Üí you define the rules.
    When the request comes in, Pydantic reads the JSON, validates it, parses it into a User object, or rejects it with errors.
    Your function only ever receives clean, validated data.

You give the types, and then its like javascript.
